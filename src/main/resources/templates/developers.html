✦ API

Travel intelligence, as an endpoint

Query the same evidence-backed safety data that powers RoamSafe: city risk scores and a live alert feed, over a simple JSON API. Structured, not scraped.

Request an API key
briefing.py 
# Get the risk profile for a city
import requests

r = requests.get(
    "https://api.roamsafe.app/api/v1/risk/city/Tokyo",
    headers={"X-API-KEY": "your_key_here"},
)
data = r.json()

print(data["overallScore"])   # 0-100
print(data["riskLevel"])      # LOW | MODERATE | HIGH | CRITICAL
for a in data["latestAlerts"]:
    print(a["name"], a["severityScore"])

Structured, not scraped

Scores, risk breakdowns and alerts as clean JSON, not HTML you have to parse.

Evidence-backed

Every score is computed from real traveler reports, the same data the site shows.

Live alert feed

Poll a paginated global feed of the newest approved reports as they land.

MCP server

Native Model Context Protocol access for AI agents. On the roadmap.

Available endpoints
GET /api/v1/risk/city/{city} Safety score, risk breakdown, summary and latest alerts for a city
GET /api/v1/alerts/feed?page=0&size=20 Paginated global feed of the newest approved reports

All requests require an X-API-KEY header. More endpoints (webhooks, neighborhood detail, MCP) are on the roadmap.
